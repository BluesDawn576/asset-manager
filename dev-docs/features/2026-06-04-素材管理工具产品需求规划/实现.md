## 目标

本轮实现要交付可运行的 Windows MVP 骨架，先验证最有风险的 Windows 文件交互，再落地素材库复制入库、本地数据库、基础预览和最小插件系统。MVP 只覆盖 Windows 11，外部投递先验证资源管理器和 QQ，首批素材类型为图片、视频、音频和文本片段。

## 实现原则

- Windows Shell/OLE 交互集中在 `Infrastructure.Windows`，UI 和插件不直接操作底层 Win32 细节。
- 素材入库默认复制到素材库目录，不只记录原路径。
- 素材库根目录中唯一自动生成的管理目录是 `.asset-manager`。
- 数据库、缩略图、临时文件、日志和库内管理状态必须放入 `.asset-manager`。
- 插件只通过 SDK 契约和宿主服务访问素材、文件能力和 UI slot。
- 每个阶段都要有可手动验证的交付物，不把插件系统和素材库主体实现混在第一阶段。

- [ ] 记录 QQ 投递成功条件和失败条件，作为后续回归测试清单。

### Phase 2: 素材库 MVP

- [ ] 实现素材库创建流程，选择或创建素材库根目录。
- [ ] 创建 `.asset-manager` 管理目录。
- [ ] 初始化 `.asset-manager/asset-manager.db`。
- [ ] 定义素材表、标签表、素材标签关系表、搜索索引表和库配置表。
- [ ] 实现文件和文件夹导入，默认复制文件到素材库目录。
- [ ] 实现同名冲突处理，MVP 默认采用自动重命名策略，例如 `name (1).ext`。
- [ ] 实现素材列表、标签编辑、收藏和基础筛选。
- [ ] 实现图片、视频、音频和文本片段的基础预览。
- [ ] 实现 SQLite FTS5 搜索，覆盖文件名、标签、备注和基础元数据。
- [ ] 确认素材库根目录除 `.asset-manager` 外不出现应用生成的管理文件。

### Phase 3: Shell 文件操作与索引同步

- [ ] 封装 `IFileOperation`，支持复制、移动、删除和重命名。
- [ ] 文件操作成功后同步更新 SQLite 索引。
- [ ] 文件操作失败时保留原索引状态，并记录错误日志。
- [ ] 实现素材文件丢失检测，支持标记失效。
- [ ] 实现从素材库复制到剪贴板后粘贴到资源管理器。
- [ ] 实现从资源管理器复制文件后粘贴入库。
- [ ] 建立手动回归场景：跨盘复制、同名冲突、权限失败、大文件拖放。

### Phase 4: 插件 MVP

- [ ] 创建 `AssetManager.Plugin.Abstractions`，定义文件类型、预览、动作、UI slot 等稳定契约。
- [ ] 创建 `AssetManager.Plugin.Sdk`，提供默认基类、属性标注和插件构建器。
- [ ] 创建 `dotnet new am-plugin` 模板。
- [ ] 实现插件包目录结构和 `plugin.json` 读取。
- [ ] 通过 `AssemblyLoadContext` 加载可信本地插件。
- [ ] 实现 UI slot registry，支持工具栏、右键菜单、预览区、详情栏和操作面板。
- [ ] 使用 Blazor Hybrid 承载插件 Razor 面板。
- [ ] 编写一个示例插件，扩展一种非内置文件类型的预览和操作面板。

### Phase 5: 插件管理与诊断

- [ ] 实现插件安装、启用、禁用和卸载。
- [ ] 显示插件 ID、名称、版本、来源、权限声明和加载状态。
- [ ] 插件加载失败时显示诊断信息，并写入 `.asset-manager/logs/`。
- [ ] 插件预览或动作失败时只影响当前插件能力，不影响主程序。
- [ ] 建立进程外插件 worker 原型，为不可信插件隔离做准备。

## 回滚方案

开发阶段每个 Phase 保持可单独回退。若 Windows 文件交互 POC 无法稳定通过，暂停后续素材库和插件实现，优先收敛 `Infrastructure.Windows`。若插件 UI 承载不稳定，先保留 WPF 原生 slot 和插件后台能力，延后复杂 Razor 面板。若素材库复制入库策略出现数据一致性问题，保留原始导入源路径记录，并通过数据库迁移修复已入库数据。
