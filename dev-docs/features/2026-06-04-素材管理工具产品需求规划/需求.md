# 素材管理工具产品需求规划

> 创建日期: 2026-06-04 11:13
> 最后更新: 2026-06-04 11:45
> 作者: Adsicmes
> 状态: 已确认

## 背景

项目计划开发一款仅面向 Windows 的素材管理工具。核心使用场景是管理本地素材文件，并像资源管理器一样与 Windows 文件系统、剪贴板、拖放和外部软件进行高频交互。工具需要支持将素材拖入、拖出、复制、粘贴到资源管理器，也需要能将素材投递到 QQ 等社交软件。

同时，素材格式、预览方式和处理流程天然会不断扩展。主程序不应把所有格式和处理逻辑写死，而应提供强扩展性的插件系统，让插件扩展文件类型识别、预览、元数据提取、缩略图生成、批处理、导出和 UI 面板。

## 目标

构建一个 Windows 本地优先的素材管理工具，满足素材收集、分类、搜索、预览、处理和外部投递的日常工作流。系统应通过插件机制持续扩展新文件类型和新处理能力，同时保证插件编写门槛低、主程序稳定、Windows 文件交互可靠。

## 用户故事

作为素材使用者，我希望能把资源管理器中的文件拖入素材库，以便快速收集和归类素材。

作为素材使用者，我希望能从素材库中直接拖出文件到资源管理器、项目目录或 QQ 聊天窗口，以便快速使用素材。

作为素材使用者，我希望能通过标签、分类、文件类型和全文搜索找到素材，以便减少重复查找和手动整理成本。

作为素材使用者，我希望在选中素材时看到预览、关键元数据和可用操作，以便不用打开外部软件也能判断素材是否可用。

作为插件作者，我希望通过简单的 C# + Razor 模板编写插件，以便扩展新的文件预览、处理动作和操作面板。

作为插件作者，我希望能向主界面的预览区、详情栏、工具栏、右键菜单和设置页注册 UI 元素，以便让插件能力自然融入宿主工作流。

## 功能描述

### 核心功能

- [ ] 素材库管理: 支持导入本地文件和文件夹，默认将素材复制到素材库目录，记录文件路径、名称、类型、大小、创建时间、修改时间、标签、备注和插件元数据。
- [ ] 分类与标签: 支持手动标签、收藏、集合、按文件类型筛选、按目录来源筛选。
- [ ] 搜索: 支持按文件名、标签、备注、扩展名、元数据全文检索。
- [ ] 预览: 首批支持图片、视频、音频和文本片段基础预览；未知格式可由插件扩展预览。
- [ ] 缩略图: 支持后台生成缩略图，并为插件提供缩略图生成接口。
- [ ] Windows 拖放: 支持资源管理器拖入、从素材库拖出到资源管理器、项目目录和 QQ 等外部应用。
- [ ] 复制粘贴: 支持将素材复制到剪贴板，再粘贴到资源管理器或外部应用；支持从资源管理器复制后粘贴入库。
- [ ] Shell 文件操作: 支持复制、移动、删除、重命名，并同步更新本地索引。
- [ ] 外部投递: 支持将真实本地文件或临时物化文件投递给 QQ 等外部软件。
- [ ] 插件系统: 支持插件扩展文件类型识别、元数据提取、缩略图、预览、单项操作、批处理、导出、设置页和 UI 面板。
- [ ] UI 扩展: 插件可向宿主暴露的 UI slot 注册按钮、菜单、详情分区、预览面板、操作面板、搜索过滤器和状态元素。
- [ ] 插件管理: 支持安装、启用、禁用、卸载插件，显示插件权限、版本、来源和加载状态。
- [ ] 后台任务: 支持素材扫描、索引更新、缩略图生成、插件处理任务和失败重试。
- [ ] 日志与诊断: 支持查看文件操作失败、插件加载失败、索引异常和后台任务错误。

### MVP 范围

- [ ] 支持 Windows 11 本地文件素材管理。
- [ ] 首批必须支持图片、视频、音频和文本片段。
- [ ] 支持文件夹导入、拖入导入、标签、搜索、图片/视频/音频/文本片段基础预览。
- [ ] 入库默认复制素材到素材库目录，不只记录原路径。
- [ ] 素材库数据库放在素材库根目录的 `.asset-manager` 目录中。
- [ ] 除素材文件本身和 `.asset-manager` 目录外，不向素材库文件夹写入其他应用生成文件。
- [ ] 支持从素材库拖出真实文件到资源管理器和 QQ。
- [ ] 支持复制素材到剪贴板，并粘贴到资源管理器。
- [ ] 支持一个最小插件 SDK，插件可扩展一种新文件类型的预览和一个操作面板。
- [ ] 支持插件安装、启用、禁用和基础权限展示。

### 素材库结构

素材库采用“文件 + 本地数据库”的结构：

```text
素材库根目录/
├── .asset-manager/
│   ├── asset-manager.db
│   ├── thumbnails/
│   ├── temp/
│   └── logs/
└── 素材文件和用户创建的目录
```

`.asset-manager` 是主程序唯一允许在素材库根目录中自动创建的管理目录。数据库、缩略图缓存、临时物化文件、日志和插件生成的库内状态如需保存在素材库内，必须放入 `.asset-manager`。除该目录外，系统不得在素材库文件夹中写入隐藏文件、索引文件、配置文件或其他管理文件。

### 暂不纳入 MVP

- Windows 资源管理器右键菜单 Shell Extension。
- 云同步、多人协作、在线插件市场。
- 素材版权管理、团队审批流、商业授权追踪。
- 跨平台支持。
- 微信、Discord、飞书等 QQ 之外的外部软件投递验证。
- 完整虚拟文件拖放协议；MVP 优先通过临时物化文件提供真实路径。

### 待确认范围

- 是否支持 Windows 10: 待确认；当前建议主目标为 Windows 11。
- 是否需要素材去重和内容哈希: 待确认。
- 是否需要文件夹监控自动同步: 待确认。
- 是否需要插件签名、插件市场和在线更新: 待确认。
- 图片、视频、音频和文本片段之外的首批内置格式是否包含字体、PSD、工程文件、压缩包: 待确认。

### 边界条件

- 外部应用只接受真实文件路径时，系统必须先把虚拟素材、远端素材或插件生成内容物化为临时文件。
- 入库复制过程中如果出现同名文件，系统需要提供可预测的冲突处理策略，例如重命名、覆盖、跳过或取消；MVP 默认策略待实现规划确认。
- 素材库目录中除 `.asset-manager` 外不能写入应用生成的管理文件；所有库内数据库、缓存、临时文件和日志必须约束在 `.asset-manager` 下。
- 插件不得直接修改宿主窗口、WPF VisualTree 或内部 ViewModel；UI 扩展必须通过宿主 UI slot registry。
- 不可信插件不得直接加载到主进程；应通过独立 worker 进程和受控 RPC 调用宿主能力。
- 插件访问文件必须通过宿主授权服务，不能默认获得全磁盘读写权限。
- 大文件复制、跨盘移动、权限受限目录和网络路径失败时，需要给出明确错误信息，并避免索引状态与文件系统状态不一致。
- 素材原文件被移动、删除或重命名后，系统需要标记失效、重新定位或提示用户修复。

## 非功能需求

- 性能: 素材列表滚动、搜索和基础预览不应被后台扫描、缩略图生成或插件处理阻塞。
- 稳定性: 插件崩溃、预览失败或处理失败不应导致主程序崩溃。
- 易用性: 常用操作应接近资源管理器体验，拖放和复制粘贴行为要符合 Windows 用户预期。
- 插件易开发: 最小插件可通过 `dotnet new am-plugin` 创建，并用少量 C# + Razor 代码声明文件类型、预览和操作面板。
- 安全性: 插件需要能力声明；不可信插件默认进程外运行；插件权限需要在 UI 中可见。
- 可维护性: 主程序、Windows 集成、插件 SDK、插件宿主、索引服务和 UI slot registry 应分层。
- 可扩展性: 新文件类型、新预览方式、新处理动作和新 UI 面板不应要求修改主程序核心代码。

## 验收标准

1. 用户可以把资源管理器中的文件或文件夹拖入应用，应用将素材复制到素材库目录并完成入库。
2. 用户可以从素材库拖出真实本地文件到资源管理器和 QQ，目标应用能接收到文件。
3. 用户可以复制素材并粘贴到资源管理器。
4. 用户可以通过标签、文件名、扩展名和备注搜索素材。
5. 用户可以预览图片、视频、音频和文本片段。
6. 插件可以新增一个宿主原本不支持的文件扩展名预览。
7. 插件可以为选中素材增加右键动作或工具栏动作。
8. 插件可以在预览区、详情栏或独立操作面板注册 Razor UI。
9. 禁用插件后，该插件贡献的预览、动作和 UI 元素消失，主程序保持可用。
10. 插件崩溃或处理失败时，主程序不崩溃，并能显示可诊断的错误信息。
11. 素材库根目录中除 `.asset-manager` 和素材文件外，不出现其他应用自动生成的管理文件。

## 优先级与排期

- 优先级: P0
- 预计工期: 待实现规划确认
- 依赖项:
  - Windows 拖放和剪贴板 POC
  - SQLite 索引和基础搜索 POC
  - 插件 SDK 与 UI slot registry POC
  - WPF + Blazor Hybrid 插件面板 POC

### 建议阶段

1. Phase 1: Windows 文件交互 POC，验证资源管理器拖入、拖出、复制、粘贴和 QQ 投递。
2. Phase 2: 素材库 MVP，完成复制入库、标签、搜索、图片/视频/音频/文本片段基础预览和本地索引。
3. Phase 3: 插件 MVP，完成插件 SDK、插件模板、文件类型预览扩展和 UI slot registry。
4. Phase 4: 插件安全与管理，完成权限声明、进程外插件宿主、启用禁用、错误诊断。
